# Data Usage Report: Multi-Agent AI for Sales Engagement

This document details the datasets utilized in the Multi-Agent Sales Engagement system, outlines their underlying schemas, explains how individual AI agents consume the data, and connects the usage directly to the core problem statement.

---

## 1. Datasets and Schema Overview

The system operates on **two primary CSV datasets** generated by `data/generate_datasets.py`. These simplified files replace the legacy 5-file structure and serve as the single authoritative input for all AI agents. They can be uploaded via the **Smart Upload** wizard which uses Ollama to automatically map columns.

### 1.1 Leads_Data.csv
**Description:** The foundational database of all potential leads — their demographics, website behavioral metrics, and CRM stage.

| Column | Type | Description |
|---|---|---|
| `lead_id` | string | Unique identifier (e.g. `L00001`) |
| `name` | string | Full name of the lead |
| `company` | string | Lead's company name |
| `title` | string | Job title |
| `industry` | string | Industry sector |
| `region` | string | Geographic region |
| `visits` | int | Number of website visits |
| `pages_per_visit` | float | Average pages browsed per visit |
| `time_on_site` | int | Total seconds spent on the website |
| `content_downloads` | int | Number of resources/assets downloaded |
| `lead_source` | string | Lead origin (e.g. LinkedIn, Website) |
| `converted` | int | 1 if converted, 0 otherwise |
| `stage` | string | CRM pipeline stage (e.g. Prospecting, Closed Won) |
| `intent_score` | float | AI-calculated purchase intent score (0–100) |
| `status` | string | Processing status (e.g. `New`, `Ready`) |

### 1.2 Email_Logs.csv
**Description:** A comprehensive history of email interactions between sales agents and leads, including full email content and engagement metrics.

| Column | Type | Description |
|---|---|---|
| `email_id` | string | Unique email record identifier (e.g. `E00001`) |
| `lead_id` | string | Foreign key linking email to a lead in `Leads_Data.csv` |
| `email_type` | string | Type: Cold Outreach, Follow-up, Newsletter, etc. |
| `subject` | string | Full email subject line |
| `email_text` | string | Complete email body (personalized per lead) |
| `opened` | int | 1 if opened, 0 otherwise |
| `replied` | int | 1 if replied, 0 otherwise |
| `click_count` | int | Number of link clicks |
| `response_status` | string | `Replied` or `No Reply` |

## 2. Dataset Generation

Datasets are generated by `data/generate_datasets.py`, which synthesizes **25,000 leads** and **25,000 emails** using the original legacy datasets as a base template. Key characteristics:

- **Leads** are sampled from `Leads_Data.csv` (original), augmented with new clean IDs, randomized industries, CRM stages, and default agent fields.
- **Emails** are fully synthesized with **realistic B2B templates** personalized per lead's name and company — across 5 email categories — with realistic open (80%) and reply (70%) engagement distributions.
- **Relational integrity**: Every `lead_id` in `Email_Logs.csv` is guaranteed to exist in `Leads_Data.csv` (100% join success verified).

## 3. Dataset Utilization by AI Agents

The Multi-Agent architecture ensures each agent is supplied only the specific slice of data it needs:

### 🕵️ Lead Research Agent
- **Datasets Used:** `Leads_Data.csv`
- **Columns Read:** `visits`, `time_on_site`, `pages_per_visit`, `converted`, `lead_source`, `region`, `company`, `title`
- **Application:** Analyzes behavioral metrics corresponding to lead demographics to segment prospects and generate engagement recommendations.

### 🎯 Intent Qualifier Agent
- **Datasets Used:** `Leads_Data.csv`, `Email_Logs.csv`
- **Columns Read:** `lead_id`, `company`, `title`, `industry`, `visits`, `content_downloads` (leads) + `lead_id`, `opened`, `replied`, `click_count` (emails)
- **Application:** Joins email engagement history to the lead by `lead_id`, then calculates an `intent_score` (0–100) and determines a `deal_value` based on title seniority and stage. Also produces `key_signals` driving the score.

### 💌 Email Strategy Agent
- **Datasets Used:** `Leads_Data.csv`, `Email_Logs.csv`
- **Columns Read:** `name`, `company`, `title`, `region`, `intent_score`, `key_signals` (from state), `email_text`, `response_status`
- **Application:** Reviews lead identity, prior email history, and intent signals to generate a fully personalized cold outreach email with a tailored subject line and call-to-action.

### ⏰ Follow-Up Timing Agent
- **Datasets Used:** State from prior agents
- **Application:** Analyzes the current intent score and recommended approach to determine optimal follow-up timing, tone (e.g. "Soft Nudge", "Urgent"), and channel.

### 📊 CRM Logger Agent
- **Datasets Used:** State from all prior agents
- **Application:** Serializes all agent outputs into a structured MongoDB document, updating `intel`, `crm`, `activity`, and `profile` subdocuments for each lead. Also writes to the `agent_activity` collection for auditing.

## 4. Smart Upload System

The **Smart Upload** system (`/api/smart-upload`) allows any company to upload their own CSV files — regardless of column naming — and have them automatically mapped to the internal schema.

1. **Upload:** User uploads a leads CSV (and optionally an email history CSV).
2. **AI Mapping:** Ollama (`minimax-m2.5:cloud`) reads the headers and sample data, then returns a JSON column mapping (e.g. `"Contact Name" → "name"`).
3. **Review & Confirm:** The frontend wizard shows the proposed mapping for user review.
4. **Processing:** The confirmed mapping is applied, defaults are injected for any missing columns, and the full 5-agent LangGraph pipeline is launched.

> **Key benefit:** Companies do not need to reformat their existing CRM exports — the AI handles the schema translation automatically.

## 5. Relevance to the Problem Statement

The core problem is **balancing scale and personalization in outbound sales outreach**.

These datasets solve that dilemma:

1. **Preventing Hallucinations:** By grounding the Ollama LLM in dense quantitative metrics from `Leads_Data.csv` and `Email_Logs.csv`, generated emails are rooted in genuine behavioral signals rather than fabricated text.
2. **Context-Driven Automation:** The email history per lead (`Email_Logs.csv`) provides the Intent Qualifier Agent with real engagement context to evaluate, rather than simple static threshold rules.
3. **Relational Memory:** The `lead_id` foreign key between both files allows the system to build a per-lead context window — combining profile, behavior, and communication history — before any AI reasoning occurs.
